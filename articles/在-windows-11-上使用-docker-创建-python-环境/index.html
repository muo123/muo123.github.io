在Windows 11上使用Docker创建Python开发环境

使用Docker创建Python开发环境可以确保开发环境的一致性，并避免"在我机器上能运行"的问题。以下是在Windows 11上设置Python开发环境的步骤：

前提条件

  1. Windows 11系统
  2. 已安装Docker Desktop for Windows
  3. 启用WSL 2（推荐）

步骤1：安装Docker Desktop

如果尚未安装：

  1. 从Docker官网下载Docker Desktop
  2. 安装并启动Docker Desktop
  3. 确保在设置中启用WSL 2后端（Settings > General > Use WSL 2 based engine）

步骤2：创建项目目录

mkdir python-docker-project
cd python-docker-project

步骤3：创建Dockerfile

在项目目录中创建Dockerfile文件（无扩展名），内容如下：

# 使用官方Python镜像作为基础
FROM python:3.9-slim

# 设置工作目录
WORKDIR /app

# 复制当前目录内容到容器中的/app目录
COPY . /app

# 安装项目依赖
RUN pip install --no-cache-dir -r requirements.txt

# 容器启动时运行的命令
CMD ["python", "your_script.py"]

步骤4：创建requirements.txt

在同一目录下创建requirements.txt文件，列出你的Python依赖，例如：

flask==2.0.1
numpy==1.21.2
pandas==1.3.3

步骤5：创建docker-compose.yml（可选但推荐）

对于更复杂的开发环境，使用docker-compose更方便：

version: '3.8'

services:
  web:
    build: .
    ports:
      - "5000:5000"
    volumes:
      - .:/app
    environment:
      - FLASK_ENV=development
    command: python your_script.py

步骤6：构建并运行容器

# 构建镜像
docker build -t python-dev .

# 运行容器（如果使用docker-compose则运行下面的命令）
docker run -it --rm -p 5000:5000 -v ${PWD}:/app python-dev

# 或者使用docker-compose
docker-compose up

开发工作流程

  1. 使用你喜欢的编辑器（如VS Code）编辑本地文件
  2. 更改会自动同步到容器中（因为挂载了卷）
  3. 测试和调试在容器环境中进行

VS Code集成（推荐）

  1. 安装"Docker"和"Remote - Containers"扩展
  2. 打开项目文件夹
  3. 按F1，选择"Remote-Containers: Open Folder in Container"
  4. 选择你的Dockerfile或docker-compose.yml

常用命令

# 查看运行中的容器
docker ps

# 进入运行中的容器
docker exec -it <container-id> bash

# 停止容器
docker stop <container-id>

# 删除所有停止的容器
docker container prune

# 删除未使用的镜像
docker image prune

提示

  • 对于数据科学项目，可以考虑使用python:3.9-slim或jupyter/datascience-notebook镜像
  • 如果需要GUI支持，需要额外配置X11转发
  • 对于生产环境，使用多阶段构建以减少镜像大小

这样你就有了一个隔离的、可复制的Python开发环境，可以在任何支持Docker的机器上运行。
