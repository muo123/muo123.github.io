在 Windows11 使用 Docker 创建 Ubuntu 容器

2025-07-09
阅读时长: 2 分钟
浏览量：...

在 Windows 系统上使用 Docker 创建 Ubuntu 容器的步骤如下：

1. 确保 Docker 环境就绪

  • 安装 Docker Desktop
    下载地址：https://www.docker.com/products/docker-desktop
    安装后启动 Docker Desktop（系统托盘出现鲸鱼图标即表示运行成功）。

  • 验证 Docker 安装
    打开 PowerShell 或 CMD，执行：

    docker --version        # 查看 Docker 版本
docker run hello-world  # 测试基础功能

2. 拉取 Ubuntu 镜像

docker pull ubuntu:latest  # 下载最新版 Ubuntu 镜像

3. 创建并运行 Ubuntu 容器

基础交互模式（临时容器）

docker run -it --name my_ubuntu ubuntu:latest
  • -it：进入交互模式（分配伪终端）
  • --name my_ubuntu：为容器命名（可自定义）
  • 此时会直接进入容器的 Bash Shell

后台运行模式（长期容器）

docker run -d --name my_ubuntu ubuntu:latest sleep infinity
  • -d：容器在后台运行
  • sleep infinity：保持容器持续运行（避免自动退出）

4. 进入已运行的容器

docker exec -it my_ubuntu /bin/bash
  • 此命令会打开容器的 Bash Shell

5. 常用容器操作命令

命令说明
docker ps查看运行中的容器
docker ps -a查看所有容器（含已停止的）
docker stop my_ubuntu停止容器
docker start my_ubuntu启动已停止的容器
docker rm my_ubuntu删除容器
docker rmi ubuntu:latest删除镜像

6. 高级用法示例

挂载主机目录到容器

docker run -it -v C:\HostData:/container_data ubuntu:latest
  • 将 Windows 的 C:\HostData 映射到容器的 /container_data

端口映射

docker run -d -p 8080:80 --name web_server ubuntu
  • 将容器的 80 端口映射到主机的 8080 端口

保存容器修改为新镜像

docker commit my_ubuntu my_custom_ubuntu

常见问题解决

  1. Docker 无法启动

    • 确保已启用 WSL 2（Windows 10/11 必需）
    • 在 PowerShell 管理员模式运行：
      wsl --update
wsl --set-default-version 2

  2. 权限不足错误

    • 在 Docker Desktop 设置中启用 WSL Integration
    • 或以管理员身份运行 PowerShell

  3. 文件路径问题

    • Windows 路径需转换为 Linux 风格（如 /c/HostData）

通过以上步骤，您已成功在 Windows 上运行 Ubuntu 容器。接下来可以在容器内安装软件（如 apt update && apt install python3）或部署应用。
