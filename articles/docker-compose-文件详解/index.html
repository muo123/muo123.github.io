Docker Compose 文件详解

Docker Compose 是一个用于定义和运行多容器 Docker 应用程序的工具，通过 YAML 文件配置应用服务。下面详细介绍如何编写 docker-compose.yml 文件来管理 Python 开发环境。

基本结构

一个典型的 Python 开发环境 docker-compose.yml 文件包含以下部分：

version: '3.8'

services:
  web:
    build: .
    ports:
      - "5000:5000"
    volumes:
      - .:/app
    environment:
      - FLASK_ENV=development
    command: flask run --host=0.0.0.0 --port=5000

核心指令详解

1. version

version: '3.8'
  • 指定使用的 Compose 文件格式版本
  • 常用版本：‘3.8’（最新稳定版）
  • 不同版本支持的功能不同

2. services

定义要运行的各个容器服务。

服务定义示例：

services:
  web:                 # 服务名称
    build: .           # 使用当前目录的Dockerfile构建
    image: my-python-app  # 可选，指定镜像名称
    container_name: my-app  # 可选，指定容器名称

3. build

指定构建镜像的配置：

build:
  context: .           # 构建上下文路径
  dockerfile: Dockerfile  # 指定Dockerfile文件名
  args:                # 构建参数
    - PYTHON_VERSION=3.9

4. ports

端口映射：

ports:
  - "5000:5000"        # 主机端口:容器端口
  - "8000:8000"
  • 左侧为主机端口，右侧为容器端口
  • 仅指定容器端口时，主机端口随机分配："5000"

5. volumes

数据卷挂载：

volumes:
  - .:/app             # 主机路径:容器路径
  - /path/on/host:/path/in/container
  - data_volume:/data  # 使用命名卷
  • 开发时常用挂载项目目录实现代码热更新
  • 命名卷需要在顶层 volumes 部分声明

6. environment

环境变量设置：

environment:
  - FLASK_ENV=development
  - FLASK_DEBUG=1
  - DATABASE_URL=postgres://user:pass@db:5432/mydb

或使用键值对格式：

environment:
  FLASK_ENV: "development"
  FLASK_DEBUG: "1"

7. command

覆盖容器启动命令：

command: flask run --host=0.0.0.0 --port=5000

或数组格式：

command: ["flask", "run", "--host=0.0.0.0", "--port=5000"]

高级配置

1. 依赖关系 (depends_on)

services:
  web:
    depends_on:
      - db
      - redis
  db:
    image: postgres:13
  redis:
    image: redis:6
  • 控制服务启动顺序
  • 不等待依赖服务"就绪"，只是"启动"

2. 健康检查 (healthcheck)

healthcheck:
  test: ["CMD", "curl", "-f", "http://localhost:5000/health"]
  interval: 30s
  timeout: 10s
  retries: 3
  start_period: 10s

3. 资源限制 (resources)

deploy:
  resources:
    limits:
      cpus: '0.50'
      memory: 512M
    reservations:
      memory: 256M

4. 网络配置 (networks)

services:
  web:
    networks:
      - frontend
      - backend

networks:
  frontend:
  backend:

完整 Python 开发环境示例

version: '3.8'

services:
  web:
    build:
      context: .
      dockerfile: Dockerfile
      args:
        - PYTHON_VERSION=3.9
    image: my-python-app
    container_name: python-web
    ports:
      - "5000:5000"
    volumes:
      - .:/app
      - /app/.venv  # 避免将虚拟环境挂载到主机
    environment:
      - FLASK_ENV=development
      - FLASK_DEBUG=1
    depends_on:
      - redis
      - db
    restart: unless-stopped
    networks:
      - app-network

  db:
    image: postgres:13
    environment:
      POSTGRES_USER: devuser
      POSTGRES_PASSWORD: devpass
      POSTGRES_DB: devdb
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    networks:
      - app-network

  redis:
    image: redis:6
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    networks:
      - app-network

volumes:
  postgres_data:
  redis_data:

networks:
  app-network:
    driver: bridge

常用命令

# 启动所有服务（后台运行）
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看服务日志
docker-compose logs -f web

# 停止服务
docker-compose down

# 重建并启动服务
docker-compose up -d --build

# 进入运行中的容器
docker-compose exec web bash

# 运行一次性命令
docker-compose run web python manage.py migrate

开发环境优化技巧

  1. 使用覆盖文件：创建 docker-compose.override.yml 用于开发特定配置
  2. 环境变量文件：使用 .env 文件管理环境变量
  3. 缓存依赖：对 pip 缓存使用命名卷
  4. 热重载：确保正确挂载代码目录并设置开发环境变量
  5. 多阶段Compose：使用不同文件管理开发和生产配置

通过合理配置 docker-compose.yml，你可以轻松管理包含多个服务的复杂 Python 开发环境，并确保团队成员使用完全相同的环境配置。
