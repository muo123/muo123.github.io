C++ 智能指针详解

C++ 智能指针是管理动态内存的现代化工具，通过自动内存释放避免内存泄漏。它们定义在 <memory> 头文件中，遵循 RAII（资源获取即初始化）原则。以下是三种主要智能指针的详细说明和代码示例：

1. std::unique_ptr（独占所有权指针）

std::unique_ptr是一种独占所有权的智能指针。同一时间只能有一个unique_ptr指向一个给定的对象。当unique_ptr被销毁（例如离开作用域）时，它所指向的对象也会被自动删除。 特性：

  • 独占资源所有权（不可复制）
  • 支持移动语义（所有权可转移）
  • 可管理数组（unique_ptr<T[]>）
  • 自定义删除器支持
  • 零运行时开销（与裸指针性能相当）

代码示例：

#include <iostream>
#include <memory>
using namespace std;

int main() {
    // 创建 unique_ptr
    unique_ptr<int> ptr1(new int(42));
    cout << *ptr1 << endl;  // 输出: 42

    // 移动所有权（转移后 ptr1 为空）
    unique_ptr<int> ptr2 = move(ptr1);
    cout << (ptr1 ? "非空" : "空") << endl;  // 输出: 空

    // 管理数组
    unique_ptr<int[]> arr(new int[3]{1, 2, 3});
    cout << arr[1] << endl;  // 输出: 2

    // 自定义删除器（示例：文件指针）
    auto fileDeleter = [](FILE* f) { 
        if(f) fclose(f); 
        cout << "文件已关闭" << endl;
    };
    unique_ptr<FILE, decltype(fileDeleter)> filePtr(fopen("test.txt", "w"), fileDeleter);

    return 0;  // 自动释放所有资源
}

2. std::shared_ptr（共享所有权指针）

std::shared_ptr是一种共享所有权的智能指针。多个shared_ptr可以指向同一个对象，并通过引用计数来管理对象的生命周期。当最后一个shared_ptr被销毁时，对象才会被删除。 特性：

  • 多个指针共享同一对象
  • 基于引用计数（自动归零时销毁对象）
  • 支持自定义删除器
  • 线程安全（引用计数原子操作）
  • 使用 make_shared 更高效（单次内存分配）

代码示例：

#include <iostream>
#include <memory>
using namespace std;

class MyClass {
public:
    MyClass() { cout << "构造函数" << endl; }
    ~MyClass() { cout << "析构函数" << endl; }
    void greet() { cout << "Hello!" << endl; }
};

int main() {
    // 推荐创建方式
    shared_ptr<MyClass> ptr1 = make_shared<MyClass>();
    ptr1->greet();  // 输出: Hello!

    {
        // 共享所有权（引用计数+1）
        shared_ptr<MyClass> ptr2 = ptr1;
        cout << "引用计数: " << ptr1.use_count() << endl;  // 输出: 2
    }  // ptr2 析构，引用计数-1

    cout << "引用计数: " << ptr1.use_count() << endl;  // 输出: 1

    // 自定义删除器示例
    shared_ptr<int> customDel(new int[5], [](int* p) {
        delete[] p;
        cout << "自定义释放数组" << endl;
    });

    return 0;  // 自动释放所有资源（输出析构函数）
}

3. std::weak_ptr（弱引用指针）

std::weak_ptr是一种不控制对象生命周期的智能指针，它指向一个由shared_ptr管理的对象。weak_ptr不会增加引用计数，因此它不会影响对象的生命周期。主要用于解决shared_ptr的循环引用问题。 特性：

  • 解决 shared_ptr 循环引用问题
  • 不增加引用计数
  • 需通过 lock() 转为 shared_ptr 访问对象
  • 检查资源是否有效

代码示例（含循环引用解决方案）：

#include <iostream>
#include <memory>
using namespace std;

class Node {
public:
    string name;
    weak_ptr<Node> partner;  // 关键：使用 weak_ptr 打破循环
    
    Node(string n) : name(move(n)) {}
    ~Node() { cout << name << " 被销毁" << endl; }
};

int main() {
    shared_ptr<Node> alice = make_shared<Node>("Alice");
    shared_ptr<Node> bob = make_shared<Node>("Bob");

    // 建立双向关联（不会导致循环引用）
    alice->partner = bob;
    bob->partner = alice;

    // 访问对象
    if (auto ptr = alice->partner.lock()) {
        cout << "Alice 的伙伴: " << ptr->name << endl;  // 输出: Bob
    }

    cout << "Alice 引用计数: " << alice.use_count() << endl;  // 输出: 1
    return 0;  // 正确销毁两个对象
}

最佳实践

  1. 优先使用 make_shared/make_unique（C++14+）：

    auto ptr = make_shared<MyClass>();  // 替代 shared_ptr(new MyClass)
    • 单次内存分配（提升性能）
    • 避免裸指针异常导致的内存泄漏

  2. 避免混用智能指针和裸指针：

    int* raw = new int(10);
shared_ptr<int> p1(raw);
// shared_ptr<int> p2(raw); // 灾难性错误：重复释放

  3. 循环引用解决方案：

    • 使用 weak_ptr 替代 shared_ptr 作为非拥有性引用

  4. 性能敏感场景：

    • 优先使用 unique_ptr（零开销抽象）
    • 避免频繁复制 shared_ptr（原子操作开销）

  5. 资源释放验证：

    • 在删除器中添加调试输出
    • 使用 Valgrind 或 AddressSanitizer 检查内存泄漏

通过合理使用智能指针，可消除 90% 以上的内存管理错误，是现代 C++ 资源管理的核心工具。
