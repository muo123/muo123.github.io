归并排序 C++ 实现

2025-07-11
阅读时长: 3 分钟
浏览量：...

归并排序算法详解

归并排序（Merge Sort）是一种基于分治策略的排序算法。其基本思想是将一个大问题分解成若干个小问题，分别解决小问题，然后将小问题的解合并起来得到原问题的解。

定义

归并排序（Merge Sort）是一种基于分治法的稳定排序算法：

  1. 时间复杂度：O(n log n)（最优、最差和平均情况相同）
  2. 空间复杂度：O(n)（需要额外存储空间）
  3. 稳定性：稳定排序（相等元素的相对位置不变）

核心思路

  1. 分治策略：

    • 分解：将数组递归地分成两半，直到子数组长度为1
    • 解决：单个元素的数组自然有序
    • 合并：将两个有序子数组合并成一个有序数组

  2. 合并操作（关键步骤）：

    • 使用三个指针分别跟踪：
      • 左子数组起始位置
      • 右子数组起始位置
      • 临时数组的写入位置
    • 比较左右子数组元素，将较小者放入临时数组
    • 将剩余元素直接复制到临时数组末尾
    • 将排序结果从临时数组复制回原数组

代码解释

#include <iostream>
using namespace std;
#define N 1000010  // 定义最大数组长度

int q[N], tmp[N];  // q: 待排序数组, tmp: 临时存储空间

void merge_sort(int *q, int l, int r) {
    // 递归终止条件：子数组长度 <= 1
    if (l >= r) return;
    
    // 分治：计算中点（注意避免溢出）
    int mid = l + r >> 1;  // 等价于 (l + r) / 2
    
    // 递归分解
    merge_sort(q, l, mid);      // 排序左半部分
    merge_sort(q, mid + 1, r);  // 排序右半部分
    
    // 合并操作
    int i = l;       // 左子数组起始指针
    int j = mid + 1; // 右子数组起始指针
    int k = 0;       // 临时数组指针
    
    // 合并两个有序子数组
    while (i <= mid && j <= r) {
        // 稳定排序的关键：相等时取左子数组元素
        if (q[i] <= q[j]) tmp[k++] = q[i++];
        else tmp[k++] = q[j++];
    }
    
    // 处理剩余元素
    while (i <= mid) tmp[k++] = q[i++];  // 左子数组剩余
    while (j <= r) tmp[k++] = q[j++];    // 右子数组剩余
    
    // 将排序结果复制回原数组
    for (int i = l, j = 0; i <= r; i++, j++) 
        q[i] = tmp[j];
}

关键点解析

  1. 分治实现：

    • 通过递归不断将数组二分（l到mid和mid+1到r）
    • 当子数组长度为1时终止递归（l >= r）

  2. 合并过程：

    • 双指针比较：同时遍历左右子数组，选择较小元素
    • 稳定性保证：q[i] <= q[j]确保相等时左侧元素优先
    • 剩余元素处理：直接复制未遍历完的子数组

  3. 空间使用：

    • 需要额外O(n)空间的tmp数组存储中间结果
    • 结果需复制回原数组q

运行样例

输入数组：[3, 1, 4, 1, 5, 9, 2, 6]
排序过程：

1. 分解：
   [3, 1, 4, 1] 和 [5, 9, 2, 6]
   → [3, 1], [4, 1], [5, 9], [2, 6]
   → [3], [1], [4], [1], [5], [9], [2], [6]

2. 合并：
   [3] & [1] → [1, 3]
   [4] & [1] → [1, 4]
   [5] & [9] → [5, 9]
   [2] & [6] → [2, 6]
   → [1, 3] & [1, 4] → [1, 1, 3, 4]
   → [5, 9] & [2, 6] → [2, 5, 6, 9]
   → [1,1,3,4] & [2,5,6,9] → [1,1,2,3,4,5,6,9]

最终输出：[1, 1, 2, 3, 4, 5, 6, 9]

算法特点

特点说明
稳定性保持相等元素原始顺序
时间复杂度始终O(n log n)，无最坏情况
空间占用需额外O(n)空间
适用场景大数据量、需要稳定排序的场景

注：代码中mid = l + r >> 1通过位运算优化除法，效果等同于(l + r) / 2但效率更高。

总结

归并排序通过分治思想，将问题规模不断缩小，然后合并有序子数组，从而完成整个数组的排序。其时间复杂度稳定为O(n log n)，但需要O(n)的额外空间。该算法适用于各种规模的数据，尤其在链表排序中具有优势（因为链表不需要额外空间进行合并）。
