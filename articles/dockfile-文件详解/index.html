Dockerfile 编写详解

Dockerfile 是一个文本文件，包含了一系列用于构建 Docker 镜像的指令。下面我将详细介绍如何编写一个高效的 Python 开发环境 Dockerfile。

基本结构

一个典型的 Python 项目 Dockerfile 包含以下部分：

# 1. 基础镜像
FROM python:3.9-slim

# 2. 元数据 (可选)
LABEL maintainer="your.email@example.com"
LABEL version="1.0"
LABEL description="Python development environment"

# 3. 环境变量设置
ENV PYTHONDONTWRITEBYTECODE 1
ENV PYTHONUNBUFFERED 1

# 4. 工作目录设置
WORKDIR /app

# 5. 依赖安装 (先复制requirements文件，利用Docker缓存层)
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 6. 复制项目文件
COPY . .

# 7. 暴露端口 (可选)
EXPOSE 8000

# 8. 启动命令
CMD ["python", "app.py"]

指令详解

1. FROM - 指定基础镜像

FROM python:3.9-slim
  • python:3.9-slim 是官方提供的轻量级 Python 镜像
  • 其他常用变体：
    • python:3.9 - 完整版
    • python:3.9-alpine - 更小的基于 Alpine Linux 的镜像
    • python:3.9-buster - 基于 Debian Buster

2. LABEL - 添加元数据

LABEL maintainer="your.email@example.com"
LABEL version="1.0"
  • 为镜像添加元信息，便于管理
  • 可以使用多个 LABEL 指令或合并为一个：
    LABEL maintainer="your.email@example.com" \
      version="1.0" \
      description="Python development environment"

3. ENV - 设置环境变量

ENV PYTHONDONTWRITEBYTECODE 1
ENV PYTHONUNBUFFERED 1
  • PYTHONDONTWRITEBYTECODE=1：防止 Python 将 pyc 文件写入磁盘
  • PYTHONUNBUFFERED=1：确保 Python 输出直接发送到终端而不被缓冲
  • 其他常用环境变量：
    ENV FLASK_APP=app.py
ENV FLASK_ENV=development

4. WORKDIR - 设置工作目录

WORKDIR /app
  • 设置容器内的工作目录
  • 如果目录不存在会自动创建
  • 后续的 RUN、CMD、COPY 等指令都会在这个目录下执行

5. COPY 和 RUN - 复制文件和安装依赖

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
  • 最佳实践：先单独复制 requirements.txt 并安装依赖，这样可以更好地利用 Docker 的缓存层
  • --no-cache-dir：告诉 pip 不要缓存下载的包，减小镜像大小
  • 对于生产环境，可以添加 --no-deps 只安装项目直接依赖

6. COPY - 复制项目文件

COPY . .
  • 第一个 . 表示主机当前目录
  • 第二个 . 表示容器内的当前目录（由 WORKDIR 指定）
  • 可以使用 .dockerignore 文件排除不需要复制的文件和目录

7. EXPOSE - 声明端口

EXPOSE 8000
  • 声明容器运行时监听的端口
  • 这只是文档作用，实际端口映射需要在 docker run 时用 -p 参数指定

8. CMD - 指定容器启动命令

CMD ["python", "app.py"]
  • 容器启动时执行的命令
  • 使用 exec 形式（JSON 数组格式）而不是 shell 形式
  • 一个 Dockerfile 中只能有一个 CMD 指令

高级用法

多阶段构建

# 第一阶段：构建环境
FROM python:3.9 as builder

WORKDIR /app
COPY requirements.txt .
RUN pip install --user -r requirements.txt

# 第二阶段：运行环境
FROM python:3.9-slim
WORKDIR /app

# 从builder阶段复制已安装的Python包
COPY --from=builder /root/.local /root/.local
COPY . .

# 确保脚本能够找到用户安装的包
ENV PATH=/root/.local/bin:$PATH

CMD ["python", "app.py"]

使用虚拟环境

FROM python:3.9-slim

RUN python -m venv /opt/venv
ENV PATH="/opt/venv/bin:$PATH"

COPY requirements.txt .
RUN pip install -r requirements.txt

COPY . .

针对开发环境的优化

FROM python:3.9-slim

# 开发专用环境变量
ENV FLASK_ENV=development
ENV FLASK_DEBUG=1

WORKDIR /app

# 安装开发依赖
COPY requirements-dev.txt .
RUN pip install -r requirements-dev.txt

COPY . .

# 使用flask run自动重载
CMD ["flask", "run", "--host=0.0.0.0"]

最佳实践

  1. 使用特定版本标签：避免使用 latest 标签
  2. 最小化镜像：使用 -slim 或 -alpine 变体
  3. 利用缓存：将不常变化的指令放在前面
  4. 减少层数：合并 RUN 指令
    RUN apt-get update && \
    apt-get install -y --no-install-recommends \
        gcc \
        python3-dev && \
    rm -rf /var/lib/apt/lists/*
  5. 安全扫描：构建后使用 docker scan 检查漏洞
  6. 使用 .dockerignore：避免复制不必要的文件

通过以上详细的 Dockerfile 编写指南，你可以为 Python 项目创建高效、安全的开发环境容器。
