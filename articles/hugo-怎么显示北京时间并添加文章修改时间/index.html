Hugo 怎么显示北京时间，并添加文章修改时间

2025-06-13
阅读时长: 1 分钟
浏览量：...

在 Hugo 中显示北京时间并添加文章修改时间，可以通过以下方式实现：

步骤：

配置站点时区

在 hugo.toml 中设置时区为 Asia/Shanghai：

timeZone = 'Asia/Shanghai'

改变部署网站的时区/.github/workflows/deploy.yaml，根据自己网站调整，下面演示的是部署网站为 Ubuntu 的设置：

  deploy:
    runs-on: ubuntu-latest
    env:
      TZ: Asia/Shanghai # 修改

修改时间格式

在 hugo.toml 中修改 dateFormat 字段

[params.dateFormat]
published = "2006-01-02"
lastUpdated = "2006-01-02 15:04 MST"

文章时间呈现效果如下

添加文章最后修改时间

在文章的首部字段中添加 lastmod 字段：

title: "文章标题"
date: 2023-10-01T08:00:00+08:00
lastmod: 2023-10-05T14:30:00+08:00  # 手动指定修改时间

或者在 hugo.toml 中设置 frontmatter 如下：

enableGitInfo = true

# 文件修改时间配置
[frontmatter]
lastmod = [':git', 'lastmod', ':fileModTime', 'date']

最终效果如下：

可能出现文章日期一致或者不能正确显示正确修改时间，需要修改 Github Action 配置文件 .github\worflows\deploy.yaml:

    steps:
      - name: Checkout
        uses: actions/checkout@v4	
        with:
          submodules: true 
          fetch-depth: 0 
      
      - name: Disable quotePath # 关闭  core.uotePath 属性
        run: git config --global core.quotePath false

当文件名包含中文时，会使得 Git 对文件路径名称进行转义处理，导致无法正确获取 GitInfo 变量信息，将 core.quotePath 设置为 false 即可解决。
